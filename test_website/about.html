Теги починаються зі знака , за яким слідує ключове слово, і закінчується знаком>. Теги діляться на дві категорії. Початковий тег, який містить ім’я тега, відкриває дію за поданням гіпертексту, а кінцевий тег, який містить теж саме ім’я тега з попереднім йому символом - кінцевий тег не потрібен Теги не чутливі до регістру символів, тобто тег TABLE може бути записаний і як table (краще так). Теги можна «вкладати» в середину інших тегів. Оскільки мова HTML побудована на базі тегів, документи цього формату можна створювати простим розміщенням тегів навколо тексту. Порядок запису атрибутів у тегу не має значення. Значення атрибута слідує за знаком рівняння, який стоїть після імені атрибута.

Якщо значення атрибута – одне слово або число, його можна вказати безпосередньо після знаку рівняння, не виділяючи додатково. Решту значень необхідно вміщувати у одинарні або подвійні лапки, особливо якщо вони містять декілька розділених пропусками слів. * атрибути id та class розглянемо в лекції по css * при програмуванні на javascript та похідних фреймворках, використовуються спеціальні атрибути - наприклад data-test, scoped, ng-if необхідні для роботи додатку
